# Security Policy

## Secret Management
- **No Secrets in Code**: Never commit secrets, API keys, or credentials to any repository.
- **Vaulting**: Use GitHub Secrets or the enterprise-approved vault for environment variables.

## Vulnerability Reporting
- If you discover a security vulnerability, do not open a public issue.
- Email the security team at security@newgameplus.com immediately.

## Dependencies
- All repositories must have Dependabot enabled.
- Security-related dependency updates must be prioritized over feature work.
